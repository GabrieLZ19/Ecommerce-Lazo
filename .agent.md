## 🏗️ Arquitectura General

### Frontend (Next.js 14)

- **Framework:** Next.js 14 con App Router
- **Lenguaje:** TypeScript
- **UI:** TailwindCSS + shadcn/ui + Radix UI
- **Estado Global:** Zustand
- **Autenticación:** Supabase Auth (email/password, OAuth Google)
- **Base de datos:** Supabase (cliente-side)
- **Pagos:** MercadoPago SDK

**Estructura Frontend:**


## 🔐 Autenticación y Autorización

**Proveedor:** Supabase Auth
**Métodos:**

- Email + Contraseña
- OAuth (Google)

**Flujo:**

1. Usuario se registra/inicia sesión en Supabase Auth
2. Supabase retorna JWT (access_token, refresh_token)
3. Frontend almacena tokens en cliente (localStorage o cookie)
4. Backend valida JWT en auth.middleware.ts
5. Sincronización de perfil: Se llama a `/api/users/sync-profile` en backend

## 💳 Sistema de Pagos (MercadoPago)

**Estado Actual:**

- Integración básica con MercadoPago
- Creación de preferencias de pago
- Webhooks para confirmación de pagos

**Endpoints:**

- `POST /api/orders/create-preference` - Crear preferencia de pago
- `POST /api/orders/webhook/mercadopago` - Webhook para notificaciones

**Flujo de Pago:**

1. Usuario agrega productos al carrito
2. En checkout, frontend llama a backend para crear preferencia
3. MercadoPago retorna URL de pago
4. Usuario es redirigido a MercadoPago
5. Después de pagar, retorna a frontend (success/failure/pending)
6. Webhook confirma pago en backend

## 📊 Flujos Principales

### 1. Compra de Productos

```
1. Usuario navega productos (GET /api/products)
2. Filtra por categoría/precio
3. Selecciona producto (GET /api/products/:id)
4. Elige talla, color y cantidad
5. Agrega al carrito (Zustand store)
6. Procede a checkout
7. Completa dirección de envío
8. Selecciona método de envío
9. Paga con MercadoPago
10. Recibe confirmación de orden
```

### 2. Gestión de Órdenes

```
Frontend:
- GET /api/orders/my-orders → Mis órdenes
- GET /api/orders/:id → Detalle de orden

Backend:
- POST /api/orders → Crear orden
- GET /api/orders/:id → Detalle
- PATCH /api/orders/:id → Actualizar estado
```

### 3. Autenticación

```
Register: POST /api/users/register
Login: POST /api/users/login
Logout: POST /api/users/logout
Perfil: GET /api/users/profile
Actualizar: PATCH /api/users/profile
```
